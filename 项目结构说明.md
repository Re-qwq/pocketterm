## 项目概述

PocketTerm 是一个 Minecraft 网易版 (Bedrock Edition) 机器人管理平台，支持：

- 多机器人管理（创建、启动、停止、删除）
- 四种认证方式（Cookie、sauth_json 直连、Fatalder API、NeOmega/FateArk）
- 自建接入点（Go 二进制实现 RakNet + MCPE 协议）
- Web 控制面板（仪表盘、机器人管理、账号管理、插件管理、文件管理）
- WebSocket 实时通信（聊天、命令、日志）
- 插件系统（Python / Go / Java 三语言支持）
- 手机端部署（零预算免费托管）

## 架构设计

### 整体架构

```
┌──────────────┐     HTTP/WS      ┌──────────────────┐     Subprocess     ┌──────────────────┐
│   前端浏览器  │ ←──────────────→ │  Python 后端     │ ←────────────────→ │  Go 接入点二进制  │
│  (HTML/JS)   │                  │  (FastAPI)       │   stdin/stdout     │  (pocketterm_ap) │
└──────────────┘                  └──────────────────┘                    └────────┬─────────┘
                                         │                                        │
                                         │ HTTP (网易直连认证)                      │ RakNet UDP
                                         ↓                                        ↓
                                 ┌──────────────┐                    ┌──────────────────┐
                                 │  网易认证服务器 │                    │  Minecraft 游戏服务器│
                                 │  (coreobt等)  │                    │  (租赁服 19132)   │
                                 └──────────────┘                    └──────────────────┘
```

### 三层架构

1. **前端层** (`frontend/`)
   - 纯 HTML + CSS + JavaScript (无框架依赖)
   - 通过 fetch API 调用后端接口
   - WebSocket 实时接收日志、聊天、事件
   - 响应式设计，支持手机端

2. **后端层** (`backend/`)
   - FastAPI + Uvicorn
   - 负责认证、机器人管理、API 路由
   - 网易直连认证模块 (Python 实现完整认证流程)
   - 通过子进程管理 Go 接入点

3. **接入点层** (`access_point_go/`)
   - Go 语言实现
   - RakNet UDP 协议连接游戏服务器
   - MCPE (Bedrock) 协议登录
   - 与 Python 后端通过 stdin/stdout JSON 通信

## 认证方式

PocketTerm 支持四种认证方式，优先级为 Cookie > sauth_json > Fatalder API > Go 内置认证：

### 1. Cookie 认证 (`cookie`)

用户提供网易登录 Cookie 字符串，后端从中提取 `SESSION` 和 `P_INFO` 构造 sauth_json，通过网易直连认证完成登录。

- 输入：HTTP Cookie 字符串 (如 `SESSION=xxx; P_INFO=yyy`)
- 流程：解析 Cookie → 构造 PC 格式 sauth_json → PC 认证 → 租赁服连接
- 优点：最简单的认证方式，用户只需提供 Cookie

### 2. 网易直连 (`direct`)

用户提供预生成的 sauth_json，直接通过网易直连认证模块完成登录。

- 输入：sauth_json 字符串 (JSON 格式)
- 流程：PE→PC 格式转换 (如需要) → PC 认证 → 租赁服连接
- 优点：最灵活，可自定义所有认证参数

### 3. Fatalder API (`fatalder`)

通过 Fatalder 第三方认证服务器完成认证。

- 输入：Fatalder API Key
- 流程：向 Fatalder 服务器发送请求 → 获取 chainInfo + 服务器地址
- 优点：不直接接触网易服务器
- 缺点：依赖第三方服务可用性

### 4. Go 内置认证 (`fbauth`)

通过 Go 二进制内置的 FastBuilder 认证完成登录。

- 输入：FB Token / 用户名密码 (可选)
- 流程：Go 二进制直接与认证服务器通信
- 用途：回退方案，当其他认证方式均不可用时使用

## 网易直连认证协议

### 概述

网易直连认证模块 (`backend/app/auth/netease_direct/`) 是通过逆向工程 Login.Core.dll 实现的完整认证协议，直接与网易服务器通信，无需第三方中转。

### 关键技术细节

#### 加密方案 (3 套密钥)

| 方案 | 用途 | 密钥编码 | AES 模式 |
|------|------|---------|---------|
| HttpEncrypt (KEYS) | 认证服务器 (authentication-v2) | UTF-8 → 16 bytes | AES-128-CBC |
| HttpEncrypt_g79v12 (KEYS_G79V12) | 核心服务器 (authentication-otp) | Hex 解码 → 16 bytes | AES-128-CBC |
| 明文 (无加密) | login-otp, API 网关, get-client-ecdh-key, get-chain-info | - | - |

> **重要发现**: 服务器已更新，`/authentication-otp` 端点使用 G79V12 密钥集（而非旧的 KEYS），请求和响应都需用 `http_encrypt_g79v12` / `http_decrypt_g79v12`。

#### 数据格式

```
HttpEncrypt 输出格式:
[16字节 IV] [AES-CBC 加密数据] [1字节 密钥选择器]

加密数据 = 原始JSON + 16个随机ASCII字符 + 零填充至块大小

密钥选择器 = (rand(0,15) << 4) | 0x0C
```

#### ComputeDynamicToken (两套算法)

**API 网关令牌** (用于 /rental-server/* 等, Login.Core.dll 版本):
```
1. combined = MD5Hex(token) + body + "0eGsBkhl" + path.TrimEnd('?')
2. md5_hex = MD5Hex(combined)  → 32字符小写hex
3. V_1 = UTF8.GetBytes(md5_hex)  → 32字节 (hex字符的ASCII码)
4. binary = ToBinary(V_1)  → 256位
5. rotated = binary[6:] + binary[:6]  → 左旋6位
6. V_1[i] ^= int(rotated[i*8:i*8+8], 2)  → 逐字节XOR
7. result = "1" + Base64(V_1)[:16].Replace("+","m").Replace("/","o")
```

**认证服务器令牌** (用于 /authentication-v2, FastBuilder 版本):
```
1. H5Token = Base64Decode(LoginSRCToken)  → 原始字节
2. combined = ToHex(H5Token, lowercase) + body + "0eGsBkhl" + path.TrimEnd('?')
3. 步骤 2-7 同上
4. user_token = ToHex(ASCII.GetBytes(result), uppercase)  → hex编码结果
```

#### PC 认证流程 (当前使用)

1. **POST /login-otp** (明文)：发送 sauth_json，获取 `aid` + `otp_token`
2. **POST /authentication-otp** (G79V12 加密)：发送认证实体，获取 `UID` + `LoginSRCToken`
3. **POST /rental-server/query/search-by-name** (API 网关, 明文)：搜索租赁服，获取 serverID
   - `user-token`: `compute_dynamic_token(path, body, LoginSRCToken)`
   - `user-id`: UID
4. **POST /rental-server-world-enter/get** (API 网关, 明文)：进入租赁服，获取 IP:Port
5. **POST /interconn/web/common/get-client-ecdh-key** (认证服务器, 明文)：获取 ECDH 密钥对
   - 无 user-token, 无 user-id (仅需 Content-Type: application/json)
6. **POST /authentication-v2** (认证服务器, 常规 KEYS 加密)：生成 Auth V2 数据
   - `user-token`: `compute_dynamic_token_auth(path, body, H5Token)` (hex编码)
   - `user-id`: UID
   - `patchVersion`: "" (空字符串)
7. **POST /interconn/web/common/get-chain-info** (认证服务器, 明文)：获取 ChainInfo (JWT 身份链)
   - 无 user-token, 无 user-id
   - ChainInfo = [get-chain-info.entity.chain[0], auth-v2.chain[0], auth-v2.chain[1]]

#### PE 认证 (不可用)

PE 认证 (`/pe-authentication`) 需要 `PESignCount` 原生签名，该函数通过 P/Invoke 调用 `Auth.Sign.dll` 中的 `CountSign` 函数，无法在 Python 中复现。详见 [已知限制](#已知限制)。

## 部署指南

### 本地部署

```bash
# 1. 安装 Python 依赖
cd PocketTerm/backend
pip install -r requirements.txt

# 2. 启动后端
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000

# 3. 打开浏览器访问 http://localhost:8000
# 默认账号: admin / admin123
```

推荐使用免费的 Python 托管服务：

1. **Render.com** (推荐) - 免费 Web Service，支持 Docker
2. **Railway.app** - 免费额度，支持 GitHub 自动部署
3. **PythonAnywhere** - 免费 Python 托管

详细部署步骤见 `部署教程.md`。

## 使用说明

### 创建机器人

1. 登录控制面板 (默认 admin/admin123)
2. 进入「机器人管理」页面
3. 点击「创建新机器人」
4. 填写信息：
   - 服务器类型：选择「租赁服」
   - 接入点：选择「自建」(使用网易直连认证)
   - 服务器编号：填写租赁服号 (如 1895088)
   - 服务器密码：无密码留空
   - 认证方式：选择对应方式
   - 根据认证方式填写 Cookie / sauth_json / API Key
5. 点击「创建」

### 启动机器人

1. 在机器人列表中找到创建的机器人
2. 点击「启动」按钮
3. 观察日志面板，等待「已成功进入游戏」提示

### 发送命令

- 在控制台输入框中输入命令 (不含 `/`)
- 点击发送或按 Enter

### 发送聊天

- 在聊天输入框中输入消息
- 点击发送或按 Enter

## 已知限制

### PE 认证不可用

PE 认证 (`/pe-authentication`) 需要原生签名 `PESignCount`，该函数通过 P/Invoke 调用 `Auth.Sign.dll` 中的 C++ 函数 `CountSign(ptr, size, offset, rounds)`，返回 16 字节签名数据。由于是原生 DLL 依赖，无法在 Python 中复现。

**解决方案**：系统自动使用 PC 认证 (`/login-otp + /authentication-otp`)，该方式不需要原生签名。如果用户提供的 sauth_json 是 PE 格式，系统会自动转换为 PC 格式。

### Fatalder API Key 时效性

Fatalder API Key 可能过期，需要定期从 [Fatalder 官网](https://fatalder.yeah114.top/) 重新获取。

### Go 二进制平台限制

`pocketterm_ap` 二进制需要根据目标平台编译。当前版本为 Linux x86_64。其他平台需要重新编译：

```bash
cd access_point_go
GOOS=linux GOARCH=amd64 go build -o pocketterm_ap .
# Windows: GOOS=windows GOARCH=amd64 go build -o pocketterm_ap.exe .
# ARM (手机): GOOS=linux GOARCH=arm64 go build -o pocketterm_ap .
```

### 反作弊防护

为防止账号被反作弊系统检测，系统已实现以下防护：

1. **独立设备指纹**：为每个账号基于 sdkuid 生成独立的设备指纹（MAC地址、UDID、磁盘序列号、CPU型号、显卡型号等），不同账号使用不同指纹，避免被识别为机器人农场
2. **删除多余字段**：移除了官方协议中不存在的 `pcCheck` 字段
3. **统一 User-Agent**：所有请求统一使用 `WPFLauncher/0.0.0.0`
4. **重连退避**：重连延迟从 5 秒增加到 30 秒，避免快速重连触发反作弊速率限制
