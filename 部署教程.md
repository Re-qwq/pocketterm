### A1. 注册 Oracle Cloud 账号
1. 打开上面链接，点击"Start for free"
2. 填写邮箱、姓名、地址（真实信息）
3. 信用卡验证（会扣1美元预授权，立即返还，**不实际扣费**）
4. 选择"Cloud Infrastructure"
5. 注册成功后登录控制台

---

### A2. 创建永久免费实例
1. 登录后，左上角菜单 → **计算** → **实例**
2. 点击**创建实例**
3. 配置实例：
   - **名称**：PocketTerm
   - **映像和配置**：点击"编辑"
     - 映像：选择 **Canonical Ubuntu 22.04** aarch64
     - 配置：选择 **Ampere A1**（ARM），设置：
       - OCPU数：**4**
       - 内存(GB)：**24**
   - **配置SSH密钥**：
     - 选择"为我生成密钥对"
     - 点击**保存私钥**，保存到本地（重要！丢失无法登录）
     - 点击**保存公钥**也保存一下
   - 其他保持默认，点击**创建**

4. 等待实例启动（状态变为"正在运行"）

### A3. 配置防火墙（重要！）
实例创建后，需要开放端口：

1. 实例详情页，找到**主VNIC** → 点击**子网**链接
2. 点击**安全列表**（Security List）
3. 点击**添加入站规则**：
   - 源 CIDR：`0.0.0.0/0`
   - 目标端口范围：`8000`
   - 协议：TCP
   - 描述：PocketTerm Web UI
   - 点击**添加入站规则**

### A4. 连接到服务器

**Windows 用户用 PowerShell：**

```powershell
# 进入下载目录
cd $env:USERPROFILE\Downloads

# 修复私钥权限（重要！）
icacls .\ssh-key-xxxx.key /inheritance:r /grant:r "$($env:USERNAME):(R)"

# 连接服务器（ubuntu 是默认用户名）
ssh -i .\ssh-key-xxxx.key ubuntu@你的公网IP
```

---

### A5. 上传项目文件到服务器

```powershell
# 在本地电脑上执行
cd C:\path\to\PocketTerm

# 用 scp 上传
scp -i $env:USERPROFILE\Downloads\ssh-key-xxxx.key -r .\* ubuntu@你的公网IP:~/PocketTerm/
```

或者用 WinSCP（图形化工具）连接上传。

---

### A6. 在服务器上部署

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Python
sudo apt install -y python3 python3-pip python3-venv

# 进入项目目录
cd ~/PocketTerm

# 创建虚拟环境
python3 -m venv backend/venv
source backend/venv/bin/activate

# 安装依赖
pip install -r backend/requirements.txt

# 测试启动
cd backend
python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

看到 "Uvicorn running on http://0.0.0.0:8000" 就成功了！按 `Ctrl+C` 停止。

---

### A7. 配置系统服务（开机自启）

```bash
# 创建服务文件
sudo nano /etc/systemd/system/pocketterm.service
```

粘贴以下内容：

```ini
[Unit]
Description=PocketTerm Minecraft Bot Panel
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/PocketTerm/backend
Environment=PATH=/home/ubuntu/PocketTerm/backend/venv/bin
ExecStart=/home/ubuntu/PocketTerm/backend/venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

按 `Ctrl+O` 保存，`Ctrl+X` 退出。

然后启用并启动：
```bash
sudo systemctl daemon-reload
sudo systemctl enable pocketterm
sudo systemctl start pocketterm

# 查看状态
sudo systemctl status pocketterm
```

看到 `active (running)` 就成功了！

## 方案D：Docker 部署（推荐生产环境）

使用 Docker 部署 PocketTerm，环境隔离、一键启动、数据持久化、自动重启。

### D1. 前置条件

确保已安装 Docker 和 Docker Compose：

```bash
# 检查 Docker 是否安装
docker --version
docker compose version

# 如未安装，请参考官方文档：
# https://docs.docker.com/get-docker/
```

### D2. 准备环境配置

```bash
# 进入项目目录
cd ~/PocketTerm

# 从示例文件创建 .env 配置
cp .env.example .env

# 按需修改配置（可选）
nano .env
```

`.env` 文件内容说明：
- `POCKETTERM_ENV=production` — 运行环境标识
- `TZ=Asia/Shanghai` — 时区设置
- `WEB_PORT=8000` — Web 服务端口

### D3. 构建并启动

```bash
# 构建镜像并后台启动
docker compose up -d --build

# 查看容器状态
docker compose ps

# 查看实时日志
docker compose logs -f
```

看到 `PocketTerm 启动完成，等待请求` 即表示启动成功。按 `Ctrl+C` 退出日志查看（容器继续运行）。

### D4. 访问面板

打开浏览器访问：`http://你的服务器IP:8000`

默认登录：
- 用户名：`admin`
- 密码：`admin123`

**登录后请立即修改密码！**

### D5. 常用运维命令

```bash
# 停止容器
docker compose down

# 启动容器（不重新构建）
docker compose up -d

# 重新构建并启动（代码更新后）
docker compose up -d --build

# 查看容器状态
docker compose ps

# 查看日志（实时）
docker compose logs -f

# 查看日志（最近 100 行）
docker compose logs --tail 100

# 重启容器
docker compose restart

# 查看健康状态
docker inspect --format='{{.State.Health.Status}}' pocketterm
```

### D6. 数据持久化说明

Docker 部署通过命名卷 `pocketterm_data` 持久化以下数据：

| 数据 | 路径 | 说明 |
|------|------|------|
| SQLite 数据库 | `/app/backend/data/pocketterm.db` | 用户、卡密、面板数据 |
| 账号数据库 | `/app/backend/data/accounts.db` | 旧版账号数据 |
| 配置文件 | `/app/backend/data/config.yaml` | 密码哈希、JWT 密钥等 |
| 日志文件 | `/app/backend/data/pocketterm.log` | 运行日志 |
| 静态数据 | `/app/backend/data/*.json` | 版本配置、方块映射等 |

**即使容器被删除重建，以上数据也不会丢失。**

查看卷的实际存储位置：
```bash
docker volume inspect pocketterm_pocketterm_data
```

备份数据：
```bash
# 备份到当前目录
docker run --rm -v pocketterm_pocketterm_data:/data -v $(pwd):/backup \
  alpine tar czf /backup/pocketterm_backup.tar.gz -C /data .

# 恢复备份
docker run --rm -v pocketterm_pocketterm_data:/data -v $(pwd):/backup \
  alpine tar xzf /backup/pocketterm_backup.tar.gz -C /data
```

### D7. 生产环境配置说明

Docker 镜像使用 `backend/config.production.yaml` 作为配置模板，与开发环境 `config.yaml` 的主要区别：

| 配置项 | 开发环境 | 生产环境 |
|--------|----------|----------|
| `logging.level` | `INFO` | `WARNING` |
| `network.verify_ssl` | `false` | `true` |
| 安全凭据 | 固定值 | 首次启动自动生成 |
| 插件目录 | `/workspace/PocketTerm/plugins` | `/app/plugins` |

首次启动时，入口脚本会自动：
1. 从 `config.production.yaml` 生成配置文件到持久化卷
2. 生成随机的 JWT 密钥和密码哈希（默认密码 `admin123`）
3. 将静态数据文件复制到挂载卷

### D8. 插件持久化（可选）

默认情况下，插件目录被打包进镜像。如需在运行时添加/修改插件，取消注释 `docker-compose.yml` 中的插件卷挂载：

```yaml
volumes:
  - pocketterm_data:/app/backend/data
  - pocketterm_plugins:/app/plugins  # 取消注释此行
```

然后重建容器：
```bash
docker compose up -d
```

### D9. 自定义端口

如需修改端口，编辑 `.env` 文件和 `docker-compose.yml`：

```bash
# .env
WEB_PORT=9000

# docker-compose.yml
ports:
  - "9000:8000"  # 宿主机端口:容器端口
```

### D10. 更新 PocketTerm

```bash
cd ~/PocketTerm

# 拉取最新代码
git pull

# 重新构建并启动
docker compose up -d --build

# 数据会自动保留（持久化卷不受影响）
```

### D11. Docker 部署常见问题

**问题：容器启动后健康检查失败**

```bash
# 查看容器日志
docker compose logs

# 检查端口是否被占用
ss -tlnp | grep 8000

# 进入容器排查
docker compose exec pocketterm sh
```

**问题：数据库初始化失败**

```bash
# 检查数据卷权限
docker compose exec pocketterm ls -la /app/backend/data/

# 如需重置数据（谨慎！会丢失所有数据）
docker compose down -v
docker compose up -d --build
```

**问题：Go/Java 插件无法加载**

Docker 镜像基于 `python:3.11-slim`，未安装 Go 和 Java 运行时。如需使用 Go/Java 插件，请在 `Dockerfile` 中添加：

```dockerfile
# 安装 Go
RUN apt-get update && apt-get install -y golang && rm -rf /var/lib/apt/lists/*

# 安装 Java
RUN apt-get update && apt-get install -y default-jdk && rm -rf /var/lib/apt/lists/*
```

Python 插件开箱即用，无需额外配置。

3. 上传/下载/删除文件

## 常见问题排查

### 问题1：浏览器打不开面板
**检查：**
1. 服务是否启动：`sudo systemctl status pocketterm`
2. 防火墙是否开放8000端口
3. Oracle Cloud 的安全列表是否添加了入站规则
4. 试试 `curl http://localhost:8000` 在服务器上能不能访问

### 问题2：机器人连不上服务器
**可能原因：**
1. 认证服务器（Fatalder）是否正常
2. API Key 是否正确
3. 服务器号/密码是否正确
4. 接入点是否已下载
5. 看控制台日志的详细错误信息

### 问题3：忘记密码
重置方法：删除 `backend/data/` 目录下的 `config.yaml`，重启服务，密码恢复为 `admin123`

Docker 部署重置方法：
```bash
# 进入容器删除配置文件
docker compose exec pocketterm rm /app/backend/data/config.yaml
# 重启容器
docker compose restart
```

### 问题4：查看日志
```bash
# 查看服务日志
sudo journalctl -u pocketterm -f

# 重启服务
sudo systemctl restart pocketterm

# 停止服务
sudo systemctl stop pocketterm
```

Docker 部署查看日志：
```bash
# 实时日志
docker compose logs -f

# 最近 100 行
docker compose logs --tail 100
```

### 问题5：更新代码
```bash
cd ~/PocketTerm
# 上传新代码覆盖后重启
sudo systemctl restart pocketterm
```

Docker 部署更新代码：
```bash
cd ~/PocketTerm
# 重新构建并启动（数据自动保留）
docker compose up -d --build
```

### 问题6：插件加载失败
- Python 插件确保有 `__init__.py` 或 `main.py`
- Go 插件需要安装 Go 编译器：`sudo apt install golang`
- Java 插件需要安装 Java：`sudo apt install default-jdk`
- 看控制台日志的详细错误信息

### 问题7：接入点下载失败
- NeOmega 和 FateArk 是二进制程序
- 可手动从 ToolDelta 的 require_files.json 下载
- 放入 `backend/data/access_points/` 目录

## 安全建议

1. **立即修改默认密码**
2. 密码输错5次会锁定15分钟（防暴力破解）
3. 不要把8000端口暴露给公网除非你设置了强密码
4. 建议配置 HTTPS（Cloudflare 或 Nginx）
5. 定期备份 `backend/data/` 目录
